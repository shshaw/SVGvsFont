This is a basic comparison between an Inline SVG fonts and Font Icons using the Font Awesome icon set. It's not a perfect 1:1 comparison (styles, colors, icon selection), but meant to demonstrate the differences in the two methods. Run Network and performance tests and draw your own conclusions. Feel free to fork, or start a discussion in the Issues.

Initial Thoughts

  • Inline SVG adds a lot of page weight. The incomplete Font Awesome set is nearly 300kb (uncacheable since it's inline), and each icon has double the amount of markup when inserted.
  • SVG renders a lot faster, which may simply be related to the SVG being inline and the font as an external asset.
  • Font Awesome's icons are much cleaner as a font, would need some optimization to show up sharper at small sizes as SVG.
  • SVG Icons need the viewBox attribute to display at the right size. Setting a general viewBox means the icon may have extra spacing around it since some icons will be larger than others. Font icons are sized to fit the glyph.
  • With SVG stroke, outline versions of icons can share the same path as the original icon, though it may not look as sharp as a real outlined path.
  • Someone smarter than I will need to run performance testing to see how SVG Icons fair versus Font Icons.
